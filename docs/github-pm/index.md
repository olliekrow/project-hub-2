# GitHub PM Setup & Workflows
{: .no_toc }

Everything you need to understand how this org's GitHub project management is structured — from org-level projects and issue templates to labeling conventions and automation.

---

## Overview

This section documents the GitHub PM architecture built for AmericaSCORES Bay Area, prototyped in the `olliekrow` demo space before org-level rollout.

The system is built around three core principles:

- **Org-level visibility** — all projects live at the org level, not the repo level, for cross-repo reporting
- **Lean tooling** — native GitHub features only (Projects v2, Actions, issue templates) unless there's a strong case for a third-party tool
- **Demo-first** — changes are validated here before proposing to the org
