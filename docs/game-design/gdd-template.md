## About this template

This template was built for kids' mobile game projects and modeled on the "Last Light" GDD as a quality benchmark. It is maintained as a Google Doc for live collaboration and referenced here for discoverability.

{: .note }
The full editable template lives in Google Docs. This page is a structural reference.

---

## Template Structure

### 1. Game Overview
- Title, tagline, genre, target audience
- Platform, approximate length, monetization model

### 2. Design Pillars
Three to five short statements that define what the game must always feel like. Used as a decision filter throughout development.

### 3. Player Personas

Personas use card-style blocks. Each card includes:
- Name (descriptive, fictional)
- Age range
- They/them pronouns
- Play context (when, where, why they play)
- Goals and frustrations

### 4. Core Loop

Diagram or written description of the primary gameplay loop:
> Act → Feedback → Reward → Next Act

### 5. Mechanics

Broken into:
- **Core mechanics** — the verbs of the game (tap, swipe, match, etc.)
- **Secondary mechanics** — systems that add depth (progression, collection, etc.)

### 6. Progression System
- Level structure
- Unlock gates
- Difficulty curve

### 7. Art Direction
- Visual style reference (mood board link)
- Color palette
- Character design notes

### 8. Audio Direction
- Music tone and tempo
- SFX philosophy

### 9. UI/UX Guidelines
- Navigation model
- Accessibility standards (font size, contrast, tap target size)

### 10. Open Questions / Risks
Unresolved design decisions and known risks tracked here until resolved.

## Standards Reminders

{: .important }
Gender-neutral language is non-negotiable. All personas use they/them and descriptive (non-gendered) names.

{: .highlight }
Card-style persona blocks only — no side-by-side table layouts.
